# Parameter Golf: Structural Overview

This repository is dedicated to the **OpenAI Model Craft Challenge: Parameter Golf**. The goal is to train the best-performing language model (measured in bits-per-byte) that fits in a **16MB artifact** and trains in under **10 minutes** on an 8xH100 machine.

## Architecture & Core Tech

The project explores advanced, parameter-efficient architectures and training techniques:
- **SKC (Spectral Koopman Capsule)**: A recurrent/hybrid architecture for long-context efficiency.
- **TTT (Test-Time Training)**: Adapting model parameters on the sequence being evaluated.
- **Ternary Quantization**: Compressing weights to {-1, 0, 1} for aggressive budget savings.
- **Engram Memory**: A non-parametric recall module using gated n-gram hashing.
- **Muon Optimizer**: A specialized optimizer for fast convergence in small-scale models.

## Project Structure

- `train_gpt_verbose.py`: **Primary source of truth.** Contains the full model definition, training logic, and hyperparameter management. Use this for editing and exploration.
- `triton_kernels.py`: Fused custom Triton kernels for high performance (FWHT, Parallel Scan, Ternary Dequant, etc.).
- `train_gpt.py`: The final, self-contained submission script. Generated by `build_submission.py`.
- `build_submission.py`: Orchestrates the transformation from verbose source to compact submission (inlining kernels, stripping comments, etc.).
- `data/`: Dataset caching and tokenizer scripts.
- `records/`: Active experiment logs and configuration snapshots.
- `_archive/`: Archived experiments and legacy scripts to keep the workspace clean.

## Key Conventions

### Configuration (Environment Variables)
The project uses a dense environment-variable-driven configuration system in `train_gpt_verbose.py` (see the `Hyperparameters` class). Use these for runtime tuning without modifying the script code:
- `COMPETITION_PROFILE=1`: Enforce strict competition constraints.
- `MAX_WALLCLOCK_SECONDS=599.0`: Hard cap on training time.
- `ARCHITECTURE=hybrid`: Switch between model types (transformer, skc, feedback).
- `EMA_ENABLED=1`: Use Exponential Moving Average for weights.

### Code Style
- **Performance First**: Extensive use of Triton for memory-bandwidth-bound operations.
- **Fused Logic**: Look for `_fwht`, `_scan`, and `_ternary` kernels in `triton_kernels.py`.
- **DDP Ready**: Training is built for multi-GPU `torchrun` environments.

## Common Workflows

### Local Development (MLX / Metal)
```bash
python3 train_gpt_mlx.py
```

### Remote Training (CUDA / RunPod)
```bash
torchrun --standalone --nproc_per_node=8 train_gpt.py
```

### Building Submission
```bash
python3 build_submission.py
```
This regenerates `train_gpt.py` from `train_gpt_verbose.py`, ensuring all custom kernels and weights are correctly bundled.

## Secrets & Remote Management

- **RunPod API Key**: Managed via `.runpod_secret.sh` (sourced by orchestration scripts).
- **RunPod SSH Keys**: Paths to SSH keys and the `.runpod_secret.sh` file itself are intentionally tracked in `.gitignore` to prevent leakage.
- **Gitignore Patterns**: Always verify that `*.secret.sh` and `.env` files are ignored before committing.
