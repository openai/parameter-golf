# Submission Strategy — Information Warfare

## The Rule
**Never submit your best work until the final 48-72 hours.**

## Public Track (Submit Early — Get Noticed)
- Model 4: Optimized transformer with TTT
- Purpose: Get on leaderboard, show competence, get judges' attention
- Reveals: Nothing novel — looks like standard transformer optimization
- Timing: Week 1-2

## Private Track (NEVER Submit Until Endgame)
- Model 1: Codec architecture (dictionary + n-gram + neural residual)
- Model 2: Adaptive recursive transformer
- Model 3: SSM/transformer hybrid
- Model 5: Best-of-everything Frankenstein
- Purpose: Develop our real competitive advantage in secret
- Reveals: Nothing — test locally, score privately
- Timing: Weeks 1-4 development, submit April 28-30 ONLY

## The Deception
Public Model 4 tells competitors: "They're doing standard transformer optimization"
Private models are: completely different architecture category
When we drop the codec/hybrid on April 28: nobody has time to copy

## Endgame (April 28-30)
1. Submit Model 5 (or whichever private model scores best)
2. Include comprehensive README — this is the interview audition
3. Competitors see it but can't respond in 48 hours
4. Novel architecture + strong score = maximum impact

## Key Constraint
PRs are public on GitHub. Even endgame submissions are visible.
But 48 hours is not enough time to understand, implement, and beat a novel architecture.
