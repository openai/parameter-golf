# HyperNetwork for Parameter Golf

## Concept

Instead of shipping the full GPT model weights in the 16MB artifact, we ship a much smaller **hypernetwork** that *generates* the GPT weights at load time.

```
Traditional:  artifact = quantized GPT weights + code
HyperNet:     artifact = quantized HyperNet weights + code
                           │
                           ▼  (forward pass at load time)
                      Full GPT weights
```

## Architecture

### HyperNetwork Structure

```
┌─────────────────────────────────────────────────┐
│  HyperNetwork                                   │
│                                                 │
│  ┌─────────────────────────────────────┐        │
│  │  Shared Trunk (MLP)                 │        │
│  │  cond_dim → hidden → hidden → chunk │        │
│  │  (64)       (512)    (512)   (4096) │        │
│  └──────────────▲──────────────────────┘        │
│                 │                                │
│  ┌──────────────┴──────────────────────┐        │
│  │  Per-Chunk Conditioning Vectors     │        │
│  │  Each chunk gets a learned 64-dim   │        │
│  │  embedding that tells the trunk     │        │
│  │  WHICH weight chunk to generate     │        │
│  └─────────────────────────────────────┘        │
│                                                 │
│  ┌─────────────────────────────────────┐        │
│  │  Direct Parameters                  │        │
│  │  Small control tensors stored as-is │        │
│  │  (q_gain, attn_scale, mlp_scale,    │        │
│  │   resid_mix, skip_weights)          │        │
│  └─────────────────────────────────────┘        │
└─────────────────────────────────────────────────┘
```

### Weight Generation Flow

For each large weight matrix in the target GPT:

1. **Chunk** — Split the weight tensor into chunks of 4096 floats
2. **Condition** — Look up the learned conditioning vector for each chunk
3. **Generate** — Pass conditioning through the shared trunk MLP
4. **Reassemble** — Concatenate chunks and reshape to the original tensor shape

```
tok_emb.weight (1024×512 = 524,288 floats)
  → 129 chunks × 4096 floats each
  → 129 conditioning vectors (each 64-dim)
  → trunk(cond) → 129 chunks of 4096 floats
  → reshape to (1024, 512)

blocks.0.mlp.fc.weight (1024×512 = 524,288 floats)
  → same process, different conditioning vectors
  → same shared trunk
```

The **shared trunk** is the key compression mechanism — all weight chunks across all layers are generated by the same MLP, differentiated only by their conditioning vectors.

### Target GPT (Generated Model)

The target GPT matches the competition baseline architecture:

```
GPT (17.06M params)
├── tok_emb: Embedding(1024, 512)          ← generated
├── blocks: 9× Block
│   ├── attn_norm: RMSNorm
│   ├── mlp_norm: RMSNorm
│   ├── attn: CausalSelfAttention
│   │   ├── c_q: Linear(512→512)           ← generated
│   │   ├── c_k: Linear(512→256)           ← generated
│   │   ├── c_v: Linear(512→256)           ← generated
│   │   ├── proj: Linear(512→512)          ← generated
│   │   ├── q_gain: (8,)                   ← stored directly
│   │   └── rotary: RoPE (buffer, not param)
│   ├── mlp: MLP
│   │   ├── fc: Linear(512→1024)           ← generated
│   │   └── proj: Linear(1024→512)         ← generated
│   ├── attn_scale: (512,)                 ← stored directly
│   ├── mlp_scale: (512,)                  ← stored directly
│   └── resid_mix: (2, 512)                ← stored directly
├── skip_weights: (4, 512)                 ← stored directly
└── final_norm: RMSNorm (no params)
```

**Generated** = large weight matrices produced by the hypernetwork trunk
**Stored directly** = small control tensors kept as hypernetwork parameters

## Parameter Budget

| Component | Params | Purpose |
|-----------|--------|---------|
| Trunk MLP | 2,397,184 | Shared weight generator (3-layer MLP) |
| Conditioning vectors | 266,240 | Per-chunk identity (tells trunk which chunk) |
| Direct params | 20,552 | Small control tensors (scales, gains, resid_mix) |
| **HyperNet total** | **2,683,976** | **What we ship** |
| **Target GPT total** | **17,059,912** | **What gets generated** |
| **Compression ratio** | **6.36x** | |

## Artifact Size

```
HyperNet weights (int8+zlib):  1.75 MB
Code (train_gpt_hypernet.py):  0.05 MB
Total artifact:                1.80 MB   ← well under 16 MB
Headroom:                     14.20 MB  (88.7%)
```

The massive headroom means we can scale up the target GPT significantly — more layers, wider MLP, larger dim — while still fitting in 16MB.

## Configuration

All config is via environment variables in the `Hyperparameters` class:

**Target GPT shape:**
- `NUM_LAYERS` (default: 9)
- `MODEL_DIM` (default: 512)
- `MLP_MULT` (default: 2)
- `NUM_HEADS` / `NUM_KV_HEADS` (default: 8 / 4)
- `VOCAB_SIZE` (default: 1024)

**HyperNetwork shape:**
- `HNET_COND_DIM` (default: 64) — conditioning vector size
- `HNET_TRUNK_HIDDEN` (default: 512) — trunk MLP hidden dim
- `HNET_TRUNK_LAYERS` (default: 3) — trunk MLP depth
- `HNET_CHUNK_SIZE` (default: 4096) — floats per generated chunk

## Training

Gradients flow through the generated weights back into the hypernetwork:

```
input_ids → HyperNet.generate_weights() → inject into GPT → GPT.forward() → loss
                    ↑                                                          │
                    └──────────── backward() ──────────────────────────────────┘
```

Only hypernetwork parameters are optimized. The target GPT has no persistent parameters — its weights are regenerated every forward pass.

## Key Tradeoffs

**Pros:**
- Artifact is ~6x smaller than direct weight storage
- Can target a much larger GPT within the same 16MB budget
- Shared trunk forces weight structure — implicit regularization

**Cons:**
- Extra forward pass cost to generate weights each step
- Gradient path is longer (through trunk → into GPT)
- Weight quality depends on trunk expressiveness
- `torch.compile` with `fullgraph=False` (dynamic weight injection)
