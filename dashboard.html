Risques : qui peut nous doubler ?

⚠
Risque #1 : Quelqu'un copie notre MoE en derniere minute ELEVE
Notre code est public sur la PR #224. Un concurrent peut prendre le TokenRoutedMLP, le greffer sur le stack SOTA (#198 : 11L + Int6 + SmearGate + SWA) et produire des resultats avant nous.

Attenuation : Le MoE seul ne suffit pas — il faut comprendre le PID, le hybrid router, et adapter le Muon optimizer pour les tenseurs 3D. Le Complexity Framework est notre avantage : on connait l'architecture, pas eux. De plus, la communaute voit notre PR en premier — toute copie flagrante serait visible.
⚠
Risque #2 : Le MoE ne bat pas le dense MOYEN
Le SOTA dense a 1.1326 BPB. Notre MoE pourrait ne pas battre ca si la specialisation des experts n'apporte pas assez de gain. Le routing deterministe (token_id % 4) est arbitraire — le hybrid router doit apprendre vite pour compenser.

Attenuation : Meme si on ne bat pas le SOTA, un score competitif (~1.12-1.14) avec une architecture fondamentalement differente est valorise par OpenAI. Ils cherchent la creativite, pas juste le leaderboard.
⚠
Risque #3 : torch.compile crash avec notre architecture MOYEN
Le mask multiply pattern est fullgraph safe en theorie, mais les tenseurs 3D (expert weights) dans Muon et le PID dynamics n'ont pas ete testes sur H100 avec torch.compile.

Attenuation : Premier run = smoke test (200 steps). Si ca crash, on a le fallback : routing="modulo" + pas de PID = version simplifiee qui compile a coup sur.
🕑
Risque #4 : Credits RunPod en retard MOYEN
La deadline est le 30 avril 2026. Si les credits mettent plus d'une semaine, on perd du temps d'iteration. Chaque run prend 10 min mais le tuning d'hyperparams en necessite des dizaines.

Attenuation : On peut payer RunPod directement (~$2/h pour 1xH100) pour un premier smoke test en attendant les credits. 1h = $2 = 6 runs.
