## Spec 021c — Container A/B/C + blend-form test (4×H paired)

021b identified that bf16 closes ~half the gap to 019b, but ~0.0007 remained unexplained.
021c ran three arms on the same 4×H JP pod to isolate container vs blend-form effects:

- **Arm A** (commit `e93d77d`): literal Python-float α, manual-add blend — the "019b" reference
- **Arm B** (commit `8b2d791`): `nn.Parameter(requires_grad=False)`, bf16, manual-add blend
- **Arm C** (commit `d070df3`): `register_buffer`, bf16, manual-add blend

Results (all 4×H, pre-quant post-EMA):
- Arm A: **1.06927**
- Arm B: **1.06952** (+0.00024 vs A — inside ±0.003 noise band)
- Arm C: partial (killed at ~4600 steps), indistinguishable from A at step 3000

**Conclusion: container choice is irrelevant at 4×H with bf16.** Literal vs Parameter vs
register_buffer all land within ±0.00024 post-EMA. The earlier 8×H gap between 021b (buffer)
and 019b (literal) is not reproduced here. It was likely pod-lottery noise, or an 8×H-specific
NCCL graph effect, not a container quality difference.

## Commit-hash confusion — discovered mid-021c session

Partway through 021c, a critical error was found: **Arm A's commit `e93d77d` was not the
actual 019b-submission code.** It was the *pre-OOM-fix manual-add variant* — the code that
existed before the OOM crash forced switching to algebraic form. The real 019b submission,
which went to the leaderboard, is at `9517a3b` and uses the algebraic lerp form
`x_before + α*(x_new - x_before)`.

We'd been comparing 021 variants against the wrong 019b reference for multiple sessions.

**Arm A' was added** (NE-1 pod, commit `9517a3b`):
- Steps: **5065** (31 more than Arm A — algebraic form + faster NE region)
- Pre-quant EMA: **1.06883** (−0.00044 vs Arm A's 1.06927)

**Algebraic > manual-add by ~0.0005 at 4×H.** This explains part of 021b's 8×H gap to
019b: 021b inherited the manual-add form, so it was penalized ~0.0005 vs the true 019b
reference even before any other difference.

This also retroactively re-explains the 021 family's regression: two separate issues had
been conflated into one — container choice (irrelevant) and blend form (real, ~0.0005).

## TTT α bug discovered

During review of the 021 commit history, a second bug was found: **the 021 lineage branched
from 017 (commit `4dd2d63`), not from 019 (`3c3a134`)**. Spec 019 added the TTT α fix —
applying the recur_alpha blend inside `forward_ttt` (the path used during phased TTT
adaptation). The 021 branch forked before that fix, so `forward_ttt` never applied the
α blend. Every post-TTT number from the 021 family was measured on a model where TTT
adapted weights as if α=1 (identity blend), then evaluated on the actual α values.

Expected cost of the bug: TTT delta shrinks by the amount α-blending helps at TTT time.
For reference, 019's fix improved TTT delta from 017's −0.01 (buggy) to ~−0.012 (fixed),
a gain of ~0.002.

**Spec 021e** is the cumulative fix: Parameter + bf16 + algebraic lerp + TTT α fix.

## Spec 021e — All four fixes, 8×H JP: new best single-seed

021e addressed both residual issues simultaneously:
1. **Algebraic blend form** `x_before + α·(x_new − x_before)` at all 4 sites
2. **TTT α fix** — applied blend in `forward_ttt`
3. Parameter container + bf16 (same as 021d)

Result (commit `d761a22`):
- Steps: **4,863**
- Val@4000: **1.1134**
- Pre-quant EMA: **1.06944**
- Post-TTT: **1.06622** — **new best single-seed result**

Δ vs 019b original: **−0.00006** — edges it out by a whisker.
Miss vs #1736: **0.00012** — within seed std (~0.0003–0.0005).

**The per-step loss gap was eliminated.** All prior 021 variants showed +0.007–0.022 higher
train_loss vs 019b at matched steps. 021e oscillates ±0.005 — pure noise. The two bug
fixes resolved it.

TTT delta for 021e: **−0.01322** (larger than 019b's −0.01249), confirming the TTT α fix
worked as intended. The larger delta represents TTT now adapting against the correct
model (with α applied) rather than an α=1 phantom.

## What 021c–021e taught us

1. **Container (literal vs Parameter vs buffer) is irrelevant** — all produce the same
   val_bpb at matched steps and dtype.
2. **Algebraic blend > manual-add by ~0.0005** — a real, reproducible effect in bf16
   due to numerical computation differences.
3. **TTT α bug was costing ~0.002–0.003 in TTT delta** — the 021 family's post-TTT
   regression was substantially caused by this.
4. **Commit-hash verification is essential** — "019b commit" referred to two different
   code paths across branches; cost several sessions of wasted comparison.

021e at 1.06622 is the new ceiling entering the learnable-α exploration arc.
