## Specs 022 and 023 — TTT-only bets, quickly shelved

After 021e landed as new best (post-TTT 1.06622), two TTT-specific ideas were specced
as orthogonal levers not requiring new training runs:

**Spec 022 — TTT Extra Depth:** Extend the loop from 3 to 4 recurrence passes at TTT
eval time only. Training uses 3 passes; the 4th LoRA slot inits B=0 (identity). At zero
submission cost, documents that benefit from deeper recurrence can push B nonzero during
TTT adaptation. Commits landed on `exp/ttt-extra-depth`.

**Spec 023 — TTT LoRA Bias:** Add a per-document bias vector (init zeros) alongside each
A×B LoRA adapter. Low-rank corrections can't express a constant mean shift; biases give
TTT a direct adaptation channel for domain-shifted documents.

Both were specced on the same branch (`b17cc65`), both marked **SHELVED** before execution.
The reason: the learnable-α warmdown-delay variant (spec 024) was judged higher-priority
given the ongoing α-optimization work. With the deadline approaching, it made more sense
to exhaust the α arc first. 022 and 023 remain valid TTT-only experiments for post-deadline
or if the deadline push needs last-minute moves.

## Spec 024 — Learnable detached-lerp

With frozen α winning at 8×H, the question became: is there a richer blend structure
worth learning and then freezing? The first step: make the learnable version more
principled by detaching the backward-path dependency.

Standard blend: `x = x_before + α*(x_new - x_before)` carries a gradient through `x_before`
across passes. This accumulates backward overhead as pass depth increases. The detached form:

```python
x_before_det = x_before.detach()
x = x_before_det + alpha * (x_new - x_before_det)
```

Same forward computation; breaks the backward retention chain. Rationale: the model should
be deciding blend depth, not accumulating upstream gradient baggage.

Result at 4×H (commit on `exp/recur-alpha-buffer`):
- Steps: 4,975, val@4000: **1.1185**, pre-quant EMA: **1.07106**

Slightly worse than 021c's frozen diagonal (1.1177) at 4×H. Expected: learnable variants
can't beat frozen at 4×H because the learning signal (gradient per optimizer step) is
limited by small-batch statistics. At 4×H, batch size is half of 8×H, so α needs more
steps to converge to the same basin. The detached lerp itself is not meaningfully better
or worse than standard lerp at 4×H.

## Spec 024b — Cross-layer carry blend, shared across passes

The bigger structural idea: instead of each layer blending only with itself (pass k of
layer i blends with pass k-1 of layer i), blend across all three looped layers using
first-pass carry states:

```python
x = beta[i] * x_new + sum_j(alpha[i,j] * carry[j])
```

where `carry[j]` = detached first-pass output of looped layer j (j ∈ {L3, L4, L5}).

This gives each layer's second pass access to the full first-pass residual stream from
all looped layers, not just its own. 12 parameters total: 3 betas + 3×3 alpha matrix.
The carry dict is populated during a full first pass before any blending begins, so there's
no causality concern — all first-pass outputs are available before any second-pass blending.

Result at 4×H (commit `89367c5`):
- Steps: **5,017**, val@4000: **1.1196**, pre-quant EMA: **1.06960**

At first glance slightly worse than 021c (1.1177). But investigation found: the 024b pod
ran ~1.2% faster than the 021c pod (4,275K tok/s vs 4,224K tok/s). The step-count
difference (5017 vs 5004) is entirely pod-speed variance. Step-matched, the two are
indistinguishable at 4×H.

**The α converged to a striking pattern:**
```
recur_beta:  [1.597, 1.883, 1.991]   # L3, L4, L5 — stronger toward L5

alpha matrix:
  L3 row: [+0.252, −0.021, −0.012]   # mostly self, ignores cross-layer
  L4 row: [+0.067, −0.348, +0.003]   # self-subtract: pass-2 L4 anti-correlates its own pass-1
  L5 row: [+0.139, +0.241, +0.027]   # pulls L3 and L4 first-pass outputs
```

`alpha[L4,L4] = −0.348` is the headline. Pass-2 L4 actively subtracts its own pass-1
output — a de-redundancy move. The model is saying "I already did L4 once; doing it again
exactly doesn't help. Let me go the opposite direction." L5 aggregates from all three.

## Spec 024c — Cross-layer carry blend, per-pass parameterization

024b shared the blend weights across both extra passes. But pass 2 and pass 3 of layer 5
are doing structurally different things — the residual stream is at different processing
stages. Giving each pass its own beta/alpha costs 24 scalars total and removes an arbitrary
sharing constraint:

```python
x = beta[pass_off, i] * x_new + sum_j(alpha[pass_off, i, j] * carry[j])
```

Result at 4×H (commit `ba957e1`):
- Running with MAX_WALLCLOCK_SECONDS=1200 (note: 024 and 024b had been missing this cap
  from their run protocols — discovered during this spec and patched retroactively)
- 024c converged to:

```
pass 1 beta:  [0.945, 1.297, 1.625]  ← amplifies with depth (L5 strongest)
pass 2 beta:  [1.984, 1.688, 1.156]  ← reverses: L3 strongest, L5 weakest
```

The two passes learned **opposite depth profiles**. Pass 1 drives amplification toward L5;
pass 2 gives maximum weight to L3 and minimum to L5. A complementary pair.

L4 self-subtract persisted in both passes:
- pass 1: alpha[L4,L4] = −0.305
- pass 2: alpha[L4,L4] = −0.291

The self-subtract is structural, not a training artifact — it reappears regardless of
pass indexing and with or without pass-sharing.

## The 4×H reliability principle (crystallized here)

After running 024, 024b, 024c: all three learnable variants ran roughly even with or
slightly behind 021c's frozen diagonal at 4×H. This pattern prompted a key realization:

**4×H comparisons are only reliable for frozen vs frozen.** For learnable vs frozen at 4×H:
- Frozen architecture works the same from step 0 regardless of batch size
- Learnable needs sufficient gradient signal per parameter step to converge to the good basin
- At 4×H, each update sees half the tokens per step vs 8×H — the learnable variant is
  effectively "batch-starved"

This is why 021c (frozen at 017's values) reliably beats 024 (learnable detached-lerp) at
4×H, even though at 8×H the learnable variant might find a better basin. The 4×H signal
for "learnable variant quality" is unreliable; the 4×H signal for "frozen variant quality"
is reliable.

**This principle directly motivates 025b**: if we freeze 024b's converged values into a
register_buffer, we get the structural benefit of cross-layer routing baked in from step 0
— no discovery cost, reliable at 4×H.
