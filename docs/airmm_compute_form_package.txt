---

BRIEF DESCRIPTION OF APPROACH:

I'm developing Adaptive Importance Routing Memory Model (AIR-MM) for the
Parameter Golf challenge. Instead of applying uniform computation to every
token, AIR-MM estimates per-token importance and selectively scales residual
updates so the model can focus limited capacity on higher-value information.

The current v2 prototype combines:
- A learned token-importance gate (~800 params/block)
- A positional recency signal
- A lightweight learned combiner (3 params/block)

The mechanism preserves a clean baseline path (USE_IMPORTANCE_ROUTING=0)
for controlled ablation.

---

WHAT HAVE YOU TRIED SO FAR:

Working baseline plus AIR-MM v1 (learned gate only) and v2 (learned gate +
recency signal + combiner) prototypes. Tested learned importance gating,
recency signal integration, and ablations against the baseline in a fast
local CPU debug loop (128-dim, 4-layer, 10 steps). Results are stable and
directionally promising: no training divergence, non-trivial gate
distributions, and small consistent improvements from v1 to v2.
