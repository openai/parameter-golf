## New Tranche Design

- Started tranche `T-20260329-B`, the architecture necessity audit, with a first 5-run set focused on `MLP_MULT` versus `NUM_LAYERS`.
- The simple question is: now that extra depth can win, are we under-layered, or are we spending too much of the budget inside each block's MLP?
- The five planned runs are:
- `B1-E1`: `10L / MLP1 / 196608 / kv2`
- `B1-E2`: `10L / MLP3 / 196608 / kv2`
- `B1-E3`: `11L / MLP1 / 196608 / kv2`
- `B1-E4`: `9L / MLP3 / 196608 / kv2`
- `B1-E5`: `9L / MLP3 / 131072 / kv2`
- This set should tell us something simple and important:
- if thinner wins at fixed depth, the current MLP may already be oversized
- if wider wins at fixed depth, pure width deserves a follow-up tranche
- if deeper-thinner wins, the model is likely under-layered relative to its MLP size
- if width only works with smaller batch, then width may have the same step-starvation pattern that depth had
