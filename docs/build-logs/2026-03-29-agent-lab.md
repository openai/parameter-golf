## Outcome

- The operating doctrine now matches the intended research style:
- early calibration can stay mostly one-factor,
- later tranches can become multi-thesis and interaction-seeking,
- failures are valuable if they teach something,
- bold or original ideas are explicitly in-bounds,
- and 600s remains the default training budget for comparable runs.
- The current runtime baseline is stronger than the prior mar28 local anchor:
- `final_int8_ttt_lora val_bpb: 1.3990`
- `751` training steps in `600342ms`
- `peak memory allocated: 5008 MiB`
- `Total submission size int8+zlib: 12642996 bytes`
- `final_int8_zlib_roundtrip_exact val_bpb: 1.43134692`
- `final_int8_ttt_lora eval_time: 800122ms`
- First capacity branch result:
- `AL-20260329-002` tested `NUM_LAYERS 9 -> 10`
- `final_int8_ttt_lora val_bpb: 1.4113`
- `676` training steps in `600527ms`
- `Total submission size int8+zlib: 13668796 bytes`
- `final_int8_zlib_roundtrip_exact val_bpb: 1.44237041`
- conclusion: depth alone is still affordable in artifact space, but it gives back too many steps at the current batch size
- Interaction follow-up result:
- `AL-20260329-003` kept `10` layers and reduced `TRAIN_BATCH_TOKENS` to `196608`
- `final_int8_ttt_lora val_bpb: 1.3916`
- `878` training steps in `600630ms`
- `Total submission size int8+zlib: 14519031 bytes`
- `final_int8_zlib_roundtrip_exact val_bpb: 1.42590892`
- conclusion: the extra depth was not wrong, it was step-starved; giving it more optimizer steps turns it into a clear win on this stack
- Frontier follow-up result:
- `AL-20260329-004` kept `10` layers and reduced `TRAIN_BATCH_TOKENS` further to `131072`
- `final_int8_ttt_lora val_bpb: 1.3913`
- `1208` training steps in `600077ms`
- `Total submission size int8+zlib: 15824353 bytes`
- `final_int8_zlib_roundtrip_exact val_bpb: 1.42696667`
- conclusion: the frontier is flattening; much more step count only bought a tiny gain, and artifact headroom is now almost gone

## Next Action

- Stay in the same tranche but change mechanism: instead of buying more steps only through smaller batches, test whether cheaper attention such as fewer KV heads can recover speed with less batch-noise penalty.
