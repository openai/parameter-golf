This branch starts from PR 1179, with its `train_gpt.py` copied to the repo root for local iteration.
We then ported mixed-precision GPTQ from PR 1105 on top, and will append experiment results here as we run them.

On 2026-03-31, a 12-layer run with `N_INT6_LAYERS=10` finished at `final_int6_sliding_window_exact val_loss:1.87586457 val_bpb:1.11099560` with `Total submission size int6+brotli: 14512536` bytes. A follow-up 12-layer run with `N_INT6_LAYERS=20` improved to `final_int6_sliding_window_exact val_loss:1.87439804 val_bpb:1.11012704` with `Total submission size int6+brotli: 15022864` bytes, still comfortably under the 15.9 MB limit.
Also on 2026-03-31, a DR run with only repeated layer `5` MLP untied finished at `final_int8_zlib_roundtrip_exact val_loss:1.87141502 val_bpb:1.10836032` with `Total submission size int6+brotli: 15574562` bytes.
Also on 2026-03-31, fully untying repeated layers `4` and `5` improved to `final_int8_zlib_roundtrip_exact val_loss:1.87004490 val_bpb:1.10754886`, but increased `Total submission size int6+brotli` to `16309808` bytes.
Also on 2026-03-31, rerunning the same config but reducing mixed-precision quantization from `45` int6 layers to `30` brought the model back under the parameter budget at `Total submission size int6+brotli: 15679328` bytes. That run finished at `final_int6_roundtrip_exact val_loss:1.91118523 val_bpb:1.13191155`, and recovered most of the eval quality with sliding-window evaluation at `final_int6_sliding_window_exact val_loss:1.87049788 val_bpb:1.10781714` (matching `final_int8_zlib_roundtrip_exact`), but it still trails the over-budget best of `1.87004490/1.10754886`.
Also on 2026-03-31, adding a delayed repeat-only learning-rate multiplier with `REPEAT_LR_MULT=1.5` regressed training noticeably after recurrence activation; by steps `4000-5500`, train loss and validation loss were clearly worse than the baseline, so this direction looks negative and should not be repeated at `1.5x`.
Also on 2026-03-31, lowering the delayed repeat-only multiplier to `REPEAT_LR_MULT=0.5` still regressed after recurrence activation (`step:3000 train_loss:1.9855` rising to `step:3500 train_loss:2.1227`, with `step:4000 val_loss:2.0911 val_bpb:1.2385`), so the repeat-only LR split looks broadly negative in this setup and was reverted.
Also on 2026-03-31, enabling late-layer partitioned residuals / partitioned hyperconnections with `PARALLEL_RESIDUAL=1`, `PARALLEL_START_LAYER=7`, physical-layer gating, recurrence-aware GPTQ calibration, and recurrent untied-MLP Hessian fixes produced a very strong quantized result despite a slight size miss: `Serialized model int6+brotli: 16001325` bytes and `Total submission size int6+brotli: 16092283` bytes. Final metrics were `final_int6_roundtrip_exact val_loss:1.90815552 val_bpb:1.13011718`, `final_int6_sliding_window_exact val_loss:1.86678942 val_bpb:1.10562077`, matching `final_int8_zlib_roundtrip_exact`.

Qualitatively, the learned routing is asymmetric: attention still writes materially into both lanes, but MLP writes very little back into the attention lane in the deeper partitioned layers (`mlp_to_attn` is near zero for physical layers `8-10`). That suggests the useful behavior here is closer to “attention broadcasts, MLP mostly stays on the prediction lane” than to a symmetric two-lane exchange.
