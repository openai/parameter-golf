# H0 - Best-of-tail EMA checkpoint selector

## Title

Best-of-tail EMA checkpoint selector.

## Source family

F0

## Mechanism

Capture a tiny portfolio of late EMA states from one training trace, export each one with the unchanged default GPTQ recipe, and then re-export the winning snapshot as the final artifact so the deployed result actually reflects the selector outcome.

## Target surface

`train_model(...)` EMA accumulation and the `train_and_eval(...)` handoff from training into export/eval.

## Broken predicate

The final EMA checkpoint is automatically the best deployed artifact.

## Score path trace

One fixed training trace -> late EMA snapshot portfolio -> unchanged `serialize(...)` / `deserialize(...)` path for each snapshot -> `quantized_sliding_window` selector -> winning snapshot is re-exported -> canonical final quantized / sliding eval runs on the winner -> compare against `C0` and `C1`.

## Operator claim

Externalize checkpoint identity into a tiny same-trace tournament instead of assuming the final EMA state is best.

## Cheap signal

Same-trace downstream delta versus both controls, with snapshot-local pre-quant `val_bpb` used only as diagnosis.

## Preserved contracts

Same seed, same data, same EMA decay, same export recipe, same byte accounting, and the same deployed sliding scorer. The train-time budget stays fixed, but the selector intentionally spends extra post-train work on late-snapshot comparison.

## Likely failure mode

Late snapshots are too sparse or downstream-indistinguishable once paired-control spread is measured.
