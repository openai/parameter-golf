# H1 - Matrix7 plus lzma export profile

## Title

Matrix7 plus lzma export profile.

## Source family

F1

## Mechanism

Freeze the trained checkpoint, widen the matrix quantizer range into int7 with a matching clip expansion, and swap the compressor to `lzma` so the export lane spends its budget on a different entropy/size trade while keeping the same recipe threaded through both write and read paths.

## Target surface

`serialize(...)`, `deserialize(...)`, `_compress(...)`, and `gptq_mixed_quantize(...)`.

## Broken predicate

The published default GPTQ/export path is already the best downstream realization for the frozen checkpoint.

## Score path trace

Frozen final checkpoint -> fixed `matrix7_lzma` export recipe -> compressed artifact bytes -> recipe-matched `deserialize(...)` -> `quantized_sliding_window`.

## Operator claim

Reallocate export budget from the current int6-plus-brotli default toward a wider matrix range plus stronger compression, while leaving training untouched.

## Cheap signal

Paired downstream delta on the same checkpoint, plus quantized-prequant gap and artifact bytes.

## Preserved contracts

Same trained checkpoint, same validation data, same scorer, same 16MB cap, and no training-loop changes.

## Likely failure mode

The wider matrix range only burns bytes or shifts compression cost without producing a downstream-score gain.
