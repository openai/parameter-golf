# H2 - Same-checkpoint export tournament

## Title

Same-checkpoint export tournament.

## Source family

F1

## Mechanism

Freeze the post-EMA checkpoint, run a tiny two-recipe export bakeoff on that same checkpoint, and then re-export the winning recipe as the final artifact so the selector outcome becomes the deployed result.

## Target surface

`serialize(...)`, `deserialize(...)`, and `train_and_eval(...)` after the pre-quant diagnostic eval.

## Broken predicate

One default export recipe is enough to identify the best deployed artifact for a frozen checkpoint.

## Score path trace

Frozen final checkpoint -> recipe `default` versus `matrix7_lzma` export -> deserialize each artifact -> `quantized_sliding_window` selector -> winning recipe is re-exported -> canonical final quantized / sliding eval runs on that winner -> compare against `C0` and `C1`.

## Operator claim

Turn export identity into a same-checkpoint tournament instead of relying on one recipe chosen a priori.

## Cheap signal

Per-recipe downstream delta at fixed checkpoint, alongside bytes and the quantized-prequant gap for attribution.

## Preserved contracts

Same training trace, same checkpoint identity, same validation shards, same scorer, and no training-time mutations. The extra work is confined to post-train export comparison.

## Likely failure mode

The extra export pass only reorders bytes and noise, or the two recipes are downstream-indistinguishable once controls are measured.
