# H3 - Transition-band loop handoff

## Title

Transition-band loop handoff.

## Source family

F2

## Mechanism

Keep the current recurrence location, but treat loop activation as a short staged handoff: when recurrence turns on, temporarily damp the learning-rate scale for a fixed transition band instead of flipping immediately into full looped updates.

## Target surface

`enable_looping_at`, the loop-activation block inside `train_model(...)`, and the per-step learning-rate scaling in `step_fn(...)`.

## Broken predicate

One hard switch at training fraction `0.5` with no explicit transition band is already the best recurrence program for deployed score.

## Score path trace

Transition-band handoff -> changed training dynamics around recurrence onset -> EMA checkpoint -> unchanged export path -> `quantized_sliding_window`.

## Operator claim

Stage the recurrence transition rather than treating loop activation as an instantaneous binary event.

## Cheap signal

Train-loss slope and tok/s around activation, then matched-wallclock pre-quant `val_bpb` before any downstream promotion.

## Preserved contracts

Same tokenizer/data contract, same model size, same loop segment, same export path, same scorer, and no simultaneous optimizer or GPTQ rewrite.

## Likely failure mode

The transition band only delays useful looping or spends wallclock without surviving EMA and export.
