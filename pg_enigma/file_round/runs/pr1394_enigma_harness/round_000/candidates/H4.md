# H4 - Attention versus MLP bit split

## Title

Attention versus MLP bit split.

## Source family

F6

## Mechanism

Rewrite the GPTQ bit-allocation law so attention matrices get a wider int7 lane while MLP matrices keep the baseline int6 lane, instead of forcing one matrix bitwidth across all quantized weights.

## Target surface

`classify_param(...)` and `gptq_mixed_quantize(...)`.

## Broken predicate

One global matrix bitwidth is already the best deployed size-quality allocation under the 16MB cap.

## Score path trace

Frozen checkpoint -> class-specific bit allocation in GPTQ -> compressed artifact bytes -> `deserialize(...)` -> `quantized_sliding_window`.

## Operator claim

Specialize bitwidth by parameter class so the export path stops spending the same precision budget on attention and MLP tensors.

## Cheap signal

Same-checkpoint downstream delta, quantized-prequant gap, and bytes at fixed training state.

## Preserved contracts

Same checkpoint, same training budget, same validation shards, same scorer, and no training-time changes.

## Likely failure mode

The added attention precision costs bytes without moving downstream score enough to escape control spread.
