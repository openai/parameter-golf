# H5 - Category-specific clip law

## Title

Category-specific clip law.

## Source family

F6

## Mechanism

Replace the single matrix clip constant with a class-specific rule: attention weights get a wider clip, MLP weights get a tighter clip, and embeddings keep the current high-protection lane.

## Target surface

`classify_param(...)` and `gptq_mixed_quantize(...)`.

## Broken predicate

One matrix-wide row-std clip law is already the best deployed size-quality rule for all quantized tensor classes.

## Score path trace

Frozen checkpoint -> class-specific clip selection in GPTQ -> compressed artifact bytes -> `deserialize(...)` -> `quantized_sliding_window`.

## Operator claim

Break the false invariant that attention and MLP rows should share the same clip multiplier even though they drive different entropy and reconstruction regimes.

## Cheap signal

Same-checkpoint downstream delta, bytes, and the quantized-prequant gap under the unchanged training trace.

## Preserved contracts

Same checkpoint, same calibration pass, same scorer, same byte accounting rule, and no training-time changes.

## Likely failure mode

The new clip split mostly reshuffles byte entropy without enough downstream benefit to beat controls.
