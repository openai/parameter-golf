# C1 - Exact replay control

## Title

Exact replay control.

## Mechanism

Replay the same unmodified baseline lane as `C0`, including the same checkpoint identity, export recipe, and downstream scoring path.

## Intended comparison role

Secondary anchor used only to estimate control spread. If `C0` and `C1` disagree enough to cover a candidate delta, the family is unresolved rather than promoted.

## Score path trace

Baseline training run -> final EMA weights -> default `serialize(...)` / `deserialize(...)` path -> `quantized` eval -> `quantized_sliding_window`.

## Preserved contracts

Identical code path, same byte rule, same tokenizer/data contract, same scorer, and no lane-specific mechanism changes.

## Novelty guard

No novel mechanism. `C1` is a replay anchor, not an alternate baseline.

## Execution note

Run the target file verbatim a second time and compare candidates against both controls, not against one anchor alone.
