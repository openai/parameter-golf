# F0 - Late checkpoint portfolio selector

## Title

Late checkpoint portfolio on one frozen training trace.

## Mechanism family

Post-train selector family that materializes a small late-window checkpoint portfolio from one identical run, then exports each checkpoint with the unchanged default GPTQ path.

## Search level

Patch-level selector family.

## Operator family

Late EMA snapshot capture, final-vs-late checkpoint comparison, and downstream paired selection under one frozen export recipe.

## Target surface

`train_model(...)` tail behavior around EMA accumulation and the post-train handoff into `serialize(...)`.

## Broken predicate

The final EMA checkpoint plus default export is automatically the best deployed artifact.

## Why baseline truth is false

The script only applies EMA once at the end, exports one artifact, and never compares late checkpoints on the deployed `quantized_sliding_window` metric. "Final is best" is therefore assumed, not established.

## Score path trace

One baseline training trace -> capture a small late checkpoint portfolio from the same EMA trajectory -> run the unchanged serialize/deserialize path for each snapshot -> score `quantized_sliding_window` -> compare against `C0` and `C1`.

## Mutation grammar

`late_portfolio := {final_only, final+1_late, final+2_late}`  
`snapshot_rule := {fixed_train_frac_tail, fixed_step_tail}`  
`selector := downstream_score_only`

## Cheap falsifier

If the best non-final snapshot fails to beat both controls outside control spread, or if all late snapshots are downstream-indistinguishable under the frozen export recipe, the family has no continuation value.

## Preserved contracts

Same seed, same data, same wallclock budget, same EMA law, same default export recipe, same byte accounting, and the same deployed sliding-window scorer.

## Family compile instructions

Patch stage may only add late-snapshot materialization and a tiny same-trace checkpoint portfolio. Do not change training dynamics or export settings inside this family; checkpoint identity is the only moving part.

## Likely failure mode

Late checkpoints are either too sparse to expose a real selector win or are washed out by control spread once scored downstream.
