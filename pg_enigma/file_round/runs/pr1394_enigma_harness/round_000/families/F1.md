# F1 - Frozen-checkpoint export menu selector

## Title

Same-checkpoint export menu bakeoff.

## Mechanism family

Executable-now post-train selector family that freezes one trained checkpoint and varies only already-expressed export recipe choices before deserialize and sliding eval.

## Search level

Executable-now family.

## Operator family

Frozen-checkpoint export-menu sweeps over existing quantization/compression knobs with paired downstream ranking.

## Target surface

`serialize(...)`, `_compress(...)`, `deserialize(...)`, and the exposed quantization/compression hyperparameters that already sit after training.

## Broken predicate

The published default GPTQ/export path is already the best downstream realization for the frozen checkpoint.

## Why baseline truth is false

The deployed metric is measured only after Hessian collection, mixed quantization, compression, deserialize, and sliding eval. The README explicitly documents real size-quality tradeoffs, so export is an active mechanism, not a passive tail.

## Score path trace

Frozen final checkpoint -> existing export recipe variant -> quantized artifact bytes -> deserialize -> `quantized_sliding_window` -> compare to `C0` and `C1` at fixed checkpoint identity.

## Mutation grammar

`compressor := {brotli, lzma}`  
`matrix_bits := bounded menu around current int6`  
`embed_bits := bounded menu around current int8`  
`clip_sigmas := bounded menu around current defaults`  
`calibration_batches := bounded menu with fixed same-checkpoint budget`

## Cheap falsifier

If menu variants only reshuffle bytes with flat/worse downstream score, or if observed deltas stay inside paired-control spread, the family does not merit further slots.

## Preserved contracts

Same checkpoint, same training trace, same validation set, same sliding-window scorer, same 16MB cap, and no changes to training-time behavior.

## Verdict

KEEP

## Candidate budget

2

## Family compile instructions

Patch-stage candidates from this family must remain export-only and same-checkpoint. Do not mix any training-loop changes into these candidates, and keep ranking anchored to downstream score first and bytes second.

## Likely failure mode

The menu only exposes shallow size/quality tradeoffs, or calibration-heavy settings burn budget without producing a real deployed-score improvement.
