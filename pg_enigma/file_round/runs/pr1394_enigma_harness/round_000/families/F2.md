# F2 - Recurrence handoff program

## Title

Phase-conditioned loop activation family.

## Mechanism family

Transition-program family that changes when and how the repeated-depth segment becomes active, including the relation between warmup, activation trigger, and late training.

## Search level

Patch-level transition-program family.

## Operator family

Loop handoff policy, activation-trigger policy, and pre-activation warmup policy.

## Target surface

`enable_looping_at`, the loop warmup block, `base_model.looping_active`, and the encoder/decoder replay path built from `loop_start`, `loop_end`, and `num_loops`.

## Broken predicate

A single fixed loop activation at training fraction `0.5` with the current one-off warmup is already the best recurrence program for deployed score.

## Why baseline truth is false

The file hard-codes a single activation fraction and a single warmup structure even though recurrence is a phase transition that changes optimization dynamics before EMA, export, and sliding eval. That is a design choice, not a verified optimum.

## Score path trace

Handoff program -> changed training dynamics around recurrence onset -> EMA trajectory -> default export -> deserialize -> `quantized_sliding_window`.

## Mutation grammar

`activation_trigger := {fixed_frac, late_frac, event_guarded_frac}`  
`warmup_rule := {current_two_pass, shortened_loop_warmup, no_extra_loop_warmup}`  
`loop_scope := {current_segment, adjacent_segment_only_if_same-mechanism}`

## Cheap falsifier

If matched-wallclock train-loss slope, tok/s around activation, and pre-quant `val_bpb` show no directional improvement before export, or if downstream gains disappear once controls are applied, stop the family.

## Preserved contracts

Same tokenizer/data contract, same model size regime, same export path, same sliding scorer, and no simultaneous optimizer or late-path rewrites.

## Verdict

REWRITE

## Candidate budget

1

## Family compile instructions

Patch stage must compile this family as an explicit handoff program with activation-side logging. Reject candidate diffs that are only naked `ENABLE_LOOPING_AT` knob nudges without a clearer transition mechanism.

## Likely failure mode

The family collapses into noisy threshold tweaks, or recurrence-side gains are swallowed by instability before they survive EMA and export.
