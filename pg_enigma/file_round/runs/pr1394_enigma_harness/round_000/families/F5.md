# F5 - Skip-correction controller

## Title

Corrective skip-gating family.

## Mechanism family

Late-path controller family that changes how `skip_weights`, `skip_gates`, and corrective skip mixing alter decoder-side routing.

## Search level

Patch-level family.

## Operator family

Corrective-skip rewrite, gate-law rewrite, and skip-controller ablations.

## Target surface

The decoder-side corrective skip block in `forward_logits(...)`, plus the learned `skip_weights` and optional `skip_gates` tensors.

## Broken predicate

The current static corrective skip formulation and gate law are already optimal for deployed score.

## Why baseline truth is false

The baseline learns one particular corrective skip controller and never compares it against alternative skip programs, even though this path directly shapes late hidden states before quantization.

## Score path trace

Skip-controller law -> changed late representations through training -> EMA checkpoint -> export -> deserialize -> `quantized_sliding_window`.

## Mutation grammar

`skip_rule := {current corrective skip, ungated corrective skip, alternate gate law}`  
`controller_scope := {all decoder skips, late-only skips}`

## Cheap falsifier

There is no clean cheap falsifier in the current harness because skip-controller weights co-adapt with training and interact with recurrence; same-checkpoint bakeoffs are not enough to test the family honestly.

## Preserved contracts

Same architecture width/depth, same recurrence base, same export path, same scorer, and no simultaneous optimizer or XSA rewrites.

## Verdict

DROP

## Candidate budget

0

## Family compile instructions

Do not allocate patch-stage slots here this round. Revive only when the harness can log skip-controller agreement signals and isolate late-path controller changes from recurrence changes.

## Likely failure mode

The family is too entangled with co-training and looping behavior to produce a trustworthy signal under the current slate budget.
