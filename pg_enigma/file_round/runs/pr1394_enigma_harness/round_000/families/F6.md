# F6 - GPTQ clip-law and bit-allocation rewrite

## Title

Quantizer-law rewrite family.

## Mechanism family

Export-only family that changes the GPTQ law itself rather than merely selecting from the current menu, while still operating on a frozen checkpoint.

## Search level

Patch-level export family.

## Operator family

Clip-law rewrites, category-specific bit allocation rewrites, and calibration-target rewrites inside the GPTQ path.

## Target surface

`collect_hessians(...)`, `gptq_quantize_weight(...)`, and `gptq_mixed_quantize(...)`.

## Broken predicate

`64` calibration batches, fixed `12.85`/`20.0` clip sigmas, and the current int6/int8 allocation are already the best deployed size-quality point under the 16MB cap.

## Why baseline truth is false

The README explicitly frames bitwidth and clip width as a compressed-size and quality tradeoff, and the deployed metric is evaluated only after this GPTQ path runs. The current law is therefore one strong recipe, not a proved optimum.

## Score path trace

Frozen checkpoint -> Hessian collection -> GPTQ law rewrite -> compressed artifact bytes -> deserialize -> `quantized_sliding_window` with the same scorer and size cap.

## Mutation grammar

`clip_law := {global row-std law, category-specific row-std law, layer-class-specific law}`  
`bit_allocation := {current int6/int8 split, alternate matrix/embed split, selective class split}`  
`calibration_target := {current module set, narrowed export-relevant module set}`

## Cheap falsifier

If same-checkpoint downstream score, quantized-prequant gap, and bytes do not move in a favorable direction together, or if improvements vanish inside paired-control spread, stop the family.

## Preserved contracts

Same trained checkpoint, same training budget, same validation shards, same sliding-window scorer, same byte accounting rule, and no training-time changes.

KEEP

## Family compile instructions

Patch-stage candidates from this family must remain frozen-checkpoint export-only and must touch only the GPTQ path. Keep calibration/export budget isolated so the family is ranked on artifact quality, not on whether training happened to finish earlier.

## Likely failure mode

Quantizer rewrites mostly reshuffle size-quality tradeoffs without a clean downstream win, or they quietly become calibration-budget hacks instead of better export laws.
