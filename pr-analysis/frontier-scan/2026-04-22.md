## Novel levers surfaced

### LoRA Warm-Start A (eval-time) — PRs #1765, #1767

**Mechanism:** In phased TTT, `BatchedLinearLoRA.reset()` currently re-randomizes both A and B between batches. Warm-start A changes this: only B is zeroed; A is kept from the previous batch. Since LoRA output = `(x @ A.T) @ B.T`, zeroing B means LoRA output is still zero at the start of each batch (per-document reset preserved), but A has accumulated useful feature directions from prior documents over ~780 batches. Combine with `alpha/rank` output scaling (standard LoRA trick) to enable stable higher rank (128 vs 96) and WD=1.0 to prevent A overfit.

**Δ-bucket:** ~−0.001 bpb on the non-CaseOps stack. May be similar or slightly different on our CaseOps stack (higher baseline TTT quality → less headroom).

**Feasibility:** Trivial code change (3-5 lines in `BatchedLinearLoRA`). No new dependencies. Fully legal under Issue #1017 (B zeros → LoRA output zeroed at each batch start; A keeps directions but output is causal).

**TTT absorption:** This IS a TTT improvement; it cannot be absorbed by itself. If stacked with our 021/022/023 changes, all improvements should be additive.

**Action:** Write spec. Low-cost (~$3 mini run to confirm delta; ~$24 for 8×H100 if confirmed). Composable with spec 021 buffer-α.
