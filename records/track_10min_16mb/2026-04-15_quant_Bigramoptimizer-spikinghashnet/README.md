# quant_Bigramoptimizer-spikinghashnet

This submission fixes a training bug in `train_gpt.py` for models using `BigramHashEmbedding`, and documents the full architecture used for the submission.

## Architecture

### BigramHashEmbedding
- Uses two independent hash tables:
  - `unigram_table` with shape `(hash_size, dim)`
  - `bigram_table` with shape `(hash_size, dim)`
- For each token, embedding is computed as:
  - `unigram_table[token_id % hash_size]`
  - `bigram_table[(prev_id * 31 + token_id) % hash_size]`
- This reduces embedding parameters from `vocab_size * dim` to `2 * hash_size * dim`.
- When `tie_embeddings=True`, the output projection still uses `tok_emb.weight`, which maps to `unigram_table.weight`.

### SpikingCausalSelfAttention
- Self-attention is implemented as a spiking causal attention block.
- Queries, keys, values, and gates are generated by linear projections.
- The query stream is gated and scaled by a learned `q_gain` parameter.
- The attention operation uses `F.scaled_dot_product_attention(..., is_causal=True)`.
- The module tracks a membrane-style state and detaches it between forward passes.

### Block design
- Each transformer block contains:
  - `RMSNorm` before attention and MLP
  - `SpikingCausalSelfAttention`
  - `MLP` with `LeakyReLUÂ˛` activation
  - Learned residual mixing via `resid_mix`
  - Separate scaling parameters for attention and MLP outputs
- The architecture is a modified transformer with spiking attention and hash-based token embeddings.

### Training and optimizer split
- Token embeddings are optimized separately from matrix and scalar parameters.
- `optimizer_tok` handles token embedding weights.
- `optimizer_muon` handles matrix parameters in transformer blocks.
- `optimizer_scalar` handles scalar and control parameters.



```python
token_lr = args.tied_embed_lr if args.tie_embeddings else args.embed_lr
tok_param_list = [base_model.tok_emb.weight]
if isinstance(base_model.tok_emb, BigramHashEmbedding):
    tok_param_list.append(base_model.tok_emb.bigram_table.weight)

optimizer_tok = torch.optim.Adam(
    [{"params": tok_param_list, "lr": token_lr, "base_lr": token_lr}],
    betas=(args.beta1, args.beta2), eps=args.adam_eps, fused=True,
)
```
