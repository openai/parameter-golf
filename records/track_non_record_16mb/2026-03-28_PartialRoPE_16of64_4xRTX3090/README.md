# Partial RoPE (16/64 dims) on 4xRTX3090

**val_bpb**: 1.35722
**artifact_size**: 15,470,414 bytes (under 16MB limit)
**peak_memory**: 9,650 MiB
**hardware**: 4x RTX 3090 (non-record, does not meet 8xH100 requirement)

## Method

We apply **Partial RoPE** to only 16 out of 64 attention head dimensions (25%), leaving the remaining 48 dimensions position-free. This allows the majority of attention capacity to focus on content-based matching while a small subset handles positional reasoning.

### Key changes from baseline (val_bpb=1.678)

| Change | Baseline | This run |
|--------|----------|----------|
| `train_batch_tokens` | 786,432 | 393,216 |
| `train_seq_len` | 512 | 1,024 |
| `rope_dims` | 64 (full) | 16 (partial, 25%) |
| `warmdown_iters` | 3,000 | 1,000 |
| `swa_every` | 50 | 25 |
| Magnitude pruning | 3% | 10% |

### Architecture (unchanged from baseline)

- 10 layers (5 encoder + 5 decoder with skip connections)
- 512 hidden dim, 8 heads, 4 KV heads (GQA)
- 3.0x MLP with ReLU^2 activation
- SmearGate, BigramHashEmbedding (10240 bigrams, 128 dim)
- INT5 MLP weights, INT6 attention weights, zstd-22 compression

### Why these changes help

1. **Halved batch tokens** (786K -> 393K): More gradient updates per wallclock (1303 steps vs 620), trading per-step quality for more optimization steps.
2. **Doubled sequence length** (512 -> 1024): Better long-range context modeling during training and evaluation.
3. **Partial RoPE** (16/64 dims): Frees 75% of attention dimensions from positional encoding, allowing content-only attention patterns. Only 25% of dims carry positional information.
4. **Warmdown + SWA tuning**: Shorter warmdown (1000 steps) with more frequent SWA checkpoints (every 25 steps) during the learning rate decay phase.
5. **Aggressive pruning** (10%): Zeros out smallest 10% of weights before quantization, reducing artifact size without meaningful quality loss.

## Training Details

- **Steps**: 1,303 (wallclock-limited at 600s)
- **SWA checkpoints**: averaged during warmdown phase
- **Gradient accumulation**: 2 steps
- **World size**: 4 GPUs
- **Optimizer**: Muon (matrix params) + AdamW (embeddings/scalars)
