# PR Archaeology Note

This note explains the "PR archaeology" part of this non-record submission.

"PR archaeology" is not an official Parameter Golf term.

Here it means reading a submission as a small historical / audit artifact:

- code
- logs
- README
- metadata
- stated intent
- hardware context
- non-record scope

The purpose is not to make the submission look stronger than it is.  
The purpose is to make it easier to understand later.

## Submission status

This is a non-record local smoke run.

It should not be read as:

- a leaderboard attempt
- a record claim
- an 8xH100 10-minute compliant run
- a statistically significant multi-seed result

It should be read as:

- a local end-to-end smoke pipeline
- an RTX2070 compatibility note
- an auditability artifact
- a record of what was actually run and regenerated

## Why this note exists

During review of my own submission artifacts, I found that an old log message could describe SDP backend settings incorrectly.

The run itself was not invalidated by that message, but the README, code, and log no longer told the same story.

The important distinction was:

> Do not edit the historical log by hand.  
> Fix the code that generates the log, rerun the pipeline, and replace the log with a newly generated one.

This note is included so that future readers can understand why log generation matters even for a non-record smoke run.
