### The structural framing

The blog represents a Mealy machine as three tensors:
- T: [input, state, state'] — transition tensor
- R: [input, state, output] — readout tensor
- s_0: [state] — initial distribution

Dynamics:
- y_t = einsum('x,s,xsy->y', x_t, s_t, R)
- s_{t+1} = einsum('x,s,xst->t', x_t, s_t, T)

The transition is **bilinear** in (input, state) — the new state depends on the outer product of input and current state via T.

### Comparing to our SSM zoo

A natural taxonomy of state evolution by *algebraic structure*:

| Model | Transition form | Algebraic structure | Compute |
|---|---|---|---|
| Mealy FSM | s' = T(x ⊗ s) | bilinear in (x, s); soft-discrete | O(N²) per step |
| Attention | y_t = retrieve(x_t, history) | content-addressable, no semigroup | O(L) per step (O(L²) total) |
| LTI SSM (S4D-Lin) | s' = A·s + B·x  with A fixed | linear; commutative monoid (powers of A) | O(L log L) via FFT |
| Mamba-1 selective | s' = exp(Δ_t·A)·s + Δ_t·B_t·x  with A diagonal | linear with data-dep coeffs; *commutative* (diagonals) | O(LN) sequential per step |
| Mamba-2 / SSD | scalar A per head; data-dep Δ_t and B_t,C_t | linear, scalar gate per head; *most-commutative* | O(L) chunkwise via matmul |
| GLA (data-dep diagonal α) | per-element data-dep gate | linear, diagonal-commutative | O(L) chunkwise |

**Mathematical insight (this is the core thing I learned from the walk)**: Mamba-2's choice of scalar A per head isn't a capacity sacrifice — it's a deliberate *structural simplification* that makes the dynamics maximally commutative. **Commutativity is what enables the chunkwise-matmul scan**: diagonal matrices commute, so the order of state updates can be reordered → parallel prefix-sum works. Non-commutative dynamics (general matrix A) break this, forcing sequential.

The data-dep gate exp(Δ_t·A) gives back per-token variability without breaking commutativity (since the gate is just a scalar applied uniformly). The scalar A is "we paid for parallelizability with structural restriction; we get expressivity back via Δ_t".

This is why Mamba-2 wins on MPS where Mamba-1 didn't: matmul-friendliness at the cost of a per-channel A degree of freedom we apparently don't miss at our regime.

### Why Hyena lost

0031 Hyena landed 0.155 BPB worse than S4D-Lin. The math:
- S4D-Lin kernel: k(t) = sum_n Re(2 · C_n · B_n · exp(t · A_n)) with A_n = -exp(A_log) + iπn (n=0..15) — a structured init giving 16 damped sinusoids spanning frequencies 0..15π. The Fourier basis is *free* at init.
- Hyena kernel: k(t) = MLP_out(SiLU(MLP_in(sin/cos(freqs·t + phases))))  — random Kaiming init at init, must LEARN all 16 frequencies.

Hyena has the same expressive class (FFT-conv long convs) but worse inductive prior. At 200 steps, the MLP can't discover what S4D-Lin's Vandermonde init *gives for free*. The **kernel parameterization matters as much as expressivity**, especially at short horizon.

This is a real writeup observation. Could even title a section: "Init is the lever, not parameterization."

### BigramHash as a 1-step FSM

BigramHash is essentially a 1-step memory FSM with hash-collision compression:
- State = previous token (vocab=1024 → 1024 "states", compressed via xor-hash to 4096 buckets)
- Transition: state' = current input (memoryless on previous)
- Output via R: read embedding(hash(state, input)) = embedding(bigram bucket)

This is provably equivalent to a 2-gram language model with a hash-bucketed lookup table replacing the full bigram matrix. The fact that it adds 0.005 BPB on top of the SSM hybrid means **the SSM stack didn't fully internalize 2-gram statistics** — which is interesting because 2-grams are the most basic n-gram thing.

Why might that be? Because the SSM has continuous state and gradient descent doesn't naturally find "use one of d_state dimensions to track 'last token id'." A discrete lookup tablebakes this in. **The lesson: discrete bounded memory and continuous unbounded memory are complementary, not redundant.** Both contribute.

### The DiffFSM lesson #2 — overparameterization works

The blog uses 8 candidate states for problems that need 2-3 states, and finds the right answer most of the time. Implication: continuous optimization with overparameterization implicitly regularizes toward simple solutions.

For Mamba-2: d_state=64 is overparameterization. After training, are most of the 64 dims used? Or do they collapse to a low-rank effective state? **Diagnostic**: post-train, look at the singular values of the (B, C, A_log) tensors. If only ~k carry signal, effective state dim is k.

This isn't an experiment — it's analysis on a trained model. Would strengthen the writeup if effective state dim is small. **Speculative**: I'd guess ~8-16 effective dims out of 64.
